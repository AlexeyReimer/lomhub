Главные правила
Не менять архитектуру без явного задания.
Не удалять существующий код.
Не создавать дублирующиеся компоненты.
Использовать существующую дизайн-систему.
Использовать TypeScript.
Не использовать any.
## Предпочитать Server Components.
Client Components использовать только при необходимости.
Все компоненты должны быть адаптивными.
