

PROJECT_RULES.md
Правила разработки проекта «ЛомХаб»
## Версия: 1.0
Главная цель
Создать лучший сервис поиска и сравнения цен на металлолом в России.
Любое техническое решение должно отвечать на вопрос:
Сделает ли это ЛомХаб быстрее, удобнее, проще и надежнее?
Если нет — решение не принимается.
Главные принципы
Простота важнее сложности.
Код должен быть понятным через год.
Не писать функциональность "на будущее", пока она не нужна.
Любой код должен быть переиспользуемым.
Любое изменение должно быть минимально необходимым.

## Компоненты
Перед созданием нового компонента проверить:
существует ли похожий компонент;
можно ли расширить существующий;
можно ли сделать компонент универсальным.
Запрещено создавать одинаковые кнопки, карточки и таблицы.
Работа с базой данных
Все изменения базы данных выполняются только через Prisma.
После изменения schema.prisma обязательно:
prisma generate
prisma migrate
проверить seed
Запрещено менять структуру таблиц вручную.

## Рефакторинг
Разрешается только если:
уменьшает сложность;
улучшает читаемость;
не ломает существующий функционал.
Запрещено делать большой рефакторинг одновременно с новой функциональностью.
